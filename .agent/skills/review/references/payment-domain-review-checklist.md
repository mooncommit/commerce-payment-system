# Payment Domain Review Checklist

## Order

* 주문 상태 초기값이 `PENDING` 인가
* 결제 검증 전 주문 완료 처리되지 않는가
* 주문 상품 스냅샷(상품명, 가격)이 저장되는가
* 총 주문 금액 계산이 정확한가
* 주문 소유자 검증이 있는가

## Payment

* Payment 레코드가 주문 생성 시 함께 생성되는가
* `portonePaymentId`가 unique 인가
* 클라이언트 결제 결과를 신뢰하지 않는가
* PortOne 조회 결과로 결제를 확정하는가
* 주문 금액과 PG 승인 금액을 검증하는가
* Payment 상태와 Order 상태가 분리되어 있는가
* 중복 결제 승인 요청에 안전한가

## Refund

* 전체 환불만 허용하는가
* 환불 가능 상태를 검증하는가
* 이미 환불된 결제에 대한 중복 요청을 방지하는가
* Refund 이력이 저장되는가
* 환불 후 주문 상태와 결제 상태가 일관되게 변경되는가
* 포인트 복구가 처리되는가

## Webhook

* Webhook payload만 신뢰하지 않는가
* Webhook 수신 후 PortOne 조회를 수행하는가
* Client Confirm과 동일한 결제 확정 로직을 사용하는가
* 이미 처리된 이벤트를 안전하게 무시하는가
* 멱등성이 보장되는가

## Point

* 포인트 잔액과 거래 내역이 분리되어 있는가
* 사용/적립/복구 이력이 저장되는가
* 포인트 부족 검증이 있는가
* 결제 실패 시 포인트 차감이 롤백되는가
* 환불 시 사용 포인트 복구와 적립 포인트 취소가 처리되는가
