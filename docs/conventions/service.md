# Service

## 역할

- 도메인 규칙과 상태 변경을 담당한다.
- 여러 도메인 변경이 하나의 유스케이스로 묶이면 command service 또는 facade로 조합한다.
- 외부 API 호출과 DB 트랜잭션 경계를 분리할 수 있으면 분리한다.

## 결제/환불

- 결제 확정은 PG 조회 결과와 DB 금액을 비교한 뒤 처리한다.
- 결제, 주문, 재고, 포인트처럼 함께 바뀌어야 하는 값은 하나의 트랜잭션으로 묶는다.
- 환불은 PG 취소 성공 이후 내부 상태를 변경한다.

## 멱등성

- 결제/웹훅/환불처럼 외부 재시도가 가능한 작업은 현재 상태를 먼저 확인한다.
- 이미 처리된 상태는 중복 변경하지 않는다.
